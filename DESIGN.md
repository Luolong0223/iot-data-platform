# DESIGN.md - IoT Data Platform 设计规范

## 视觉风格
- **主色调**: 深蓝-青绿（深色技术感），强调"数据驾驶舱"意象
- **辅助色**: 琥珀色（告警/警告）、翠绿（在线）、朱红（离线/严重）
- **图表库**: Chart.js 4.x
- **图标**: Bootstrap Icons
- **暗色优先**: 仪表盘/大屏使用深色背景（#0a1929 系列），运维页面支持浅色

## 设计原则
1. **数据密度**: 大屏/控制台优先信息密度，运维页面优先可读性
2. **可发现性**: 关键操作主按钮 + 图标按钮
3. **实时感**: 数字滚动、动画过渡、状态点闪烁
4. **可扩展性**: 卡片式布局，易于后续添加模块

## 关键页面
- `/dashboard` - 主控台：4 统计卡 + 趋势 + 实时流 + 告警
- `/screen` - 数据大屏：可选数据点 + 实时值 + 趋势
- `/devices` - 设备列表：状态指示 + 批量操作 + 地图
- `/hierarchy` - 项目→分组→设备树形管理
- `/alarms` - 告警中心
- `/alarm-rules` - 告警规则配置
- `/map` - 设备地图（Leaflet/百度）

## 字体
- 系统字体优先，等宽数字用于数据展示

## 不做的事
- ❌ 蓝紫渐变（AI 味）
- ❌ Inter/Roboto 等通用无衬线
- ❌ 居中对称的"卡片网格"
- ❌ 装饰性动画干扰数据阅读
